## commit template 등록 방법
# git config commit.template .commit_template.txt

[Type] Subject

# - 제목은 50자 이내
# - 대문자로 시작
# - 마침표 없이 현재형 동사 사용
# - 예시: [Feat] Add user login API
#
# [Type] 목록
# - [Feat] 기능 추가
# - [Bug] 버그 수정
# - [Refactor] 리팩토링
# - [Docs] 문서 변경
# - [Build] 빌드 관련 작업
# - [CI/CD] 자동화, CI/CD 관련 작업
# - [Chore] 기타 작업
# - [Enhance] 기능

# 한 줄 띄우기

# Body (선택사항)
# - 각 줄은 72자 이내로 작성
# - 무엇을, 왜 변경했는지 중심으로 작성
# - 어떻게 변경했는지는 부가 설명

# 예시:
# 사용자 인증 기능 개발을 위한 API를 추가함.
# JWT를 사용한 로그인 토큰을 반환하도록 구현.

# 한 줄 띄우기

# Footer (선택사항)
# - 관련 이슈가 있을 경우 아래 형식 사용
# - 키워드: Fixes, Resolves
# 예시:
# Fixes: #123, #456
# Related to: #789
